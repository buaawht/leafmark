# LeafMark V0.1.1 设计文档

日期：2026-06-20

## 1. 概述

LeafMark V0.1.1 在 V0.1.0 的本地 Markdown 编辑闭环之上，扩展成更完整的轻量本地工作区。

V0.1.1 的目标不是变成复杂知识库或 IDE，而是在保持单窗口、轻量、本地优先的前提下，补齐用户手动 QA 后提出的核心使用需求：

- App icon。
- 浅色 / 深色 / 跟随系统外观模式。
- 编辑区和预览区同步滚动。
- 更清晰的 toolbar。
- Save / Save As 合并成一个保存下拉操作。
- 左侧目录树。
- 打开整个目录。
- 文件和文件夹的新建、重命名、删除。
- 多标签编辑。
- 导出 Markdown 和 PDF。

V0.1.1 仍然不做 Git 集成、插件系统、云同步、账号、协作、复杂知识库、App Store 发布、签名公证或复杂主题市场。

## 2. 产品目标

V0.1.1 完成后，LeafMark 应该支持：

- 使用自定义 LeafMark app icon。
- 在 System / Light / Dark 三种外观模式之间切换。
- 打开目录后，在左侧显示目录树。
- 展开和折叠目录。
- 点击目录树中的文件打开或切换到对应 tab。
- 新建 Markdown 文件。
- 新建文件夹。
- 重命名文件和文件夹。
- 删除文件和文件夹到系统 Trash。
- 同时打开多个文件，每个文件对应一个 tab。
- 每个 tab 可以单独保存、关闭。
- 支持 Close Others。
- 未保存文件不会被其他打开操作覆盖。
- 编辑区和预览区同步滚动，优先做到实际可用。
- Save 和 Save As 通过一个保存控件组织。
- toolbar 按钮默认显示文字，让功能可发现。
- 导出当前文档为 Markdown。
- 导出当前文档为 PDF。

## 3. 非目标

V0.1.1 不做：

- Git 集成。
- 插件系统。
- 云同步。
- 账号。
- 多人协作。
- 全文搜索。
- 双链、知识图谱、复杂知识库管理。
- 复杂主题市场。
- App Store 发布。
- 签名、公证、安装器。
- 复杂编辑器能力，例如语法高亮、自动补全、Markdown 工具按钮栏。

## 4. 主界面

V0.1.1 主界面采用单窗口三栏结构：

- 左侧：目录树，可显示或隐藏。
- 中间：Markdown 编辑区。
- 右侧：WKWebView 预览区。
- 顶部：toolbar 和 tab bar。
- 底部：status area。

布局方向：

```text
┌──────────────────────────────────────────────────────────────┐
│ Toolbar: New / Open Folder / Save ▾ / Export ▾ / Appearance │
├──────────────────────────────────────────────────────────────┤
│ Tabs: notes.md  ideas.md  ...                               │
├───────────────┬──────────────────────┬───────────────────────┤
│ Directory     │ Editor               │ Preview               │
│ Tree          │                      │ WKWebView             │
│               │                      │                       │
├───────────────┴──────────────────────┴───────────────────────┤
│ Status: saved state / words / characters / path               │
└──────────────────────────────────────────────────────────────┘
```

目录树可以展开和关闭。关闭目录树后，编辑器和预览区占据更多空间。

## 5. App icon

LeafMark V0.1.1 需要有自己的 macOS app icon。

视觉方向：

- 轻量。
- 本地。
- Markdown。
- leaf / mark 意象。
- 适合 macOS dock 和 Finder。

实现要求：

- 生成 icon 源图。
- 输出 macOS app icon 所需尺寸。
- 接入 app bundle。
- `Info.plist` 指向 icon。
- 构建脚本把 icon 资源复制进 `.app` bundle。

第一版 icon 不要求建立复杂品牌系统。确认一个方向后接入即可。

## 6. 外观模式

LeafMark 支持三种外观模式：

- System：跟随系统。
- Light：强制浅色。
- Dark：强制深色。

默认值：System。

外观模式需要影响：

- SwiftUI 主界面。
- 编辑区。
- 预览区 HTML/CSS。

实现方向：

- 增加 `AppearancePreference`。
- 在 app 状态中保存当前选择。
- SwiftUI 使用 `preferredColorScheme`。
- `MarkdownRenderService` 或 preview 渲染入口接收外观信息，并输出对应 CSS。
- System 模式下，preview CSS 可以使用 `prefers-color-scheme`。

V0.1.1 不要求持久化到复杂配置系统；如果实现成本低，可以使用 `AppStorage` 保存。

## 7. 多标签

V0.1.1 从单文档状态升级为多文档 session。

每个 tab 对应一个 document state：

- Markdown 文本。
- baseline text。
- file URL。
- display name。
- dirty state。
- preview HTML。
- scroll state。

行为：

- 点击目录树文件，如果文件已打开，则切换到已有 tab。
- 如果文件未打开，则新建 tab。
- 新建未保存文件也以 tab 形式存在。
- 每个 tab 可以单独关闭。
- dirty tab 关闭时弹 Save / Don't Save / Cancel。
- Close Others 关闭除当前 tab 外的其他 tab，对 dirty tab 逐个处理或集中处理。
- Save 保存当前 tab。
- Save As 更新当前 tab 的 file URL 和标题。

V0.1.1 不要求 tab 可拖拽排序。

## 8. 目录树和目录导入

用户可以打开一个目录作为当前 workspace root。

目录树显示：

- 文件夹。
- 支持的文件：`.md`、`.markdown`、`.txt`。
- 可以保留其他文件不可点击或隐藏；V0.1.1 推荐隐藏不支持文件，减少噪音。

行为：

- Open Folder 选择一个目录。
- 左侧目录树加载目录结构。
- 文件夹可以展开和折叠。
- 点击支持文件打开 tab。
- 目录树可以刷新。

V0.1.1 不做全文搜索和复杂文件过滤。

## 9. 文件管理

V0.1.1 支持基础文件管理：

### 新建文件

- 在当前选中目录下创建文件。
- 如果没有选中目录，则在 workspace root 下创建。
- 默认扩展名 `.md`。
- 文件名冲突时提示用户改名，不覆盖。
- 创建成功后打开为 tab。

### 新建文件夹

- 在当前选中目录下创建文件夹。
- 如果没有选中目录，则在 workspace root 下创建。
- 文件夹名冲突时提示用户改名，不覆盖。
- 创建成功后刷新目录树。

### 重命名

- 支持重命名文件。
- 支持重命名文件夹。
- 文件名冲突时提示错误，不覆盖。
- 如果重命名的文件已在 tab 中打开，需要同步更新 tab 的 file URL 和标题。
- 如果文件有未保存修改，重命名前先走 Save / Don't Save / Cancel。
- Cancel 时不重命名。

### 删除

- 删除文件或文件夹时，移动到系统 Trash，不做永久删除。
- 删除前必须确认。
- 删除非空文件夹也必须确认。
- 如果删除目标已在 tab 中打开：
  - 没有未保存修改：关闭 tab 后删除。
  - 有未保存修改：先走 Save / Don't Save / Cancel。
  - Cancel 时不删除。
- 删除成功后刷新目录树。
- 删除失败时显示明确错误。

## 10. Toolbar

Toolbar 需要更清楚。

V0.1.1 toolbar 包含：

- New。
- Open Folder。
- Save ▾。
- Export ▾。
- Toggle Sidebar。
- Toggle Preview。
- Appearance。
- Copy Rendered。

要求：

- 按钮默认显示文字。
- Save 和 Save As 合并到 Save 下拉中。
- Export 下拉包含 Export Markdown 和 Export PDF。

仍然保持简洁，不增加 Markdown 格式按钮栏。

## 11. 同步滚动

V0.1.1 需要编辑区和预览区同步滚动。

目标：

- 编辑 Markdown 时，预览区滚动到大致对应位置。
- 用户滚动编辑区时，预览区同步到相近位置。
- 用户滚动预览区时，编辑区可以同步到相近位置；如果双向同步成本过高，优先保证编辑区到预览区。

实现方向：

- 编辑区需要能报告 scroll percentage。
- preview WebView 需要能设置和报告 scroll percentage。
- 先采用百分比同步，不追求按 Markdown block 精准定位。
- 防止两个区域互相触发无限滚动循环。

验收标准：

- 长文档中，从顶部滚到中部/底部时，另一侧大致同步。
- 不要求逐标题精确锚定。

## 12. 导出

V0.1.1 支持导出当前 tab。

导出 Markdown：

- 用户选择目标路径。
- 写出当前 Markdown source。
- 不改变当前 tab 的 file URL，除非用户明确使用 Save As。

导出 PDF：

- 使用当前 preview HTML 渲染为 PDF。
- 用户选择目标路径。
- PDF 应包含基础 Markdown 格式。
- 本地图片尽量保留。

实现方向：

- ExportService 负责导出。
- Markdown export 使用 UTF-8 写文件。
- PDF export 可使用 WKWebView PDF 能力或 AppKit printing 能力，优先选择简单稳定方案。

V0.1.1 不要求导出 Word、HTML 或图片格式。

## 13. 数据模型

需要从单个 `DocumentViewModel` 演进为 session 结构。

建议组件：

- `DocumentTab`
  - 单个打开文档状态。
- `DocumentSessionViewModel`
  - 管理 tabs、selected tab、workspace root、文件操作。
- `DirectoryTreeModel`
  - 表示目录树节点。
- `WorkspaceFileService`
  - 新建、重命名、删除、扫描目录。
- `AppearancePreference`
  - System / Light / Dark。
- `ExportService`
  - Markdown / PDF 导出。

V0.1.1 可以保留并复用现有 `DocumentViewModel` 的部分逻辑，但不应该让单个文件继续拥有全局 app 状态。

## 14. 错误处理

必须明确处理：

- 打开目录失败。
- 扫描目录失败。
- 打开文件失败。
- 保存失败。
- Save As 取消。
- 新建文件/文件夹失败。
- 文件名冲突。
- 重命名失败。
- 删除到 Trash 失败。
- PDF 导出失败。
- dirty tab 关闭被取消。

不能静默丢失用户内容。

## 15. 验收方式

自动验证：

- `swift build`。
- `scripts/build_app_bundle.sh`。
- `plutil -lint`。
- `git diff --check`。
- 如果环境支持 XCTest，运行 `swift test`。

手动 QA：

- App icon 在 Finder/Dock 显示。
- System / Light / Dark 切换有效。
- 打开目录后目录树显示。
- 目录树可展开/折叠。
- 点击文件打开 tab。
- 多 tab 不互相覆盖。
- dirty tab 关闭有确认。
- Close Others 正确处理 dirty tabs。
- 新建文件。
- 新建文件夹。
- 重命名文件。
- 重命名文件夹。
- 删除文件到 Trash。
- 删除文件夹到 Trash。
- Save 下拉包含 Save / Save As。
- Export 下拉包含 Markdown / PDF。
- Markdown 导出正确。
- PDF 导出可打开且格式基本正确。
- 长文档编辑/预览同步滚动可用。
- toolbar 按钮文字默认可见。

## 16. 实现原则

- 保持本地优先。
- 保持单窗口。
- 功能完成优先于复杂 polish。
- 文件删除必须安全，使用 Trash。
- 多标签和目录树是 V0.1.1 的主要结构变化，需要先打好状态模型。
- 不把 V0.1.1 顺手扩展成知识库或 IDE。
