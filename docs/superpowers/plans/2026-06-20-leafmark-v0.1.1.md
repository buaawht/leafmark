# LeafMark V0.1.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade LeafMark from a single-document Markdown editor into a lightweight local workspace with folder navigation, multi-tab editing, document outline, appearance modes, scroll sync, export, and a custom app icon.

**Architecture:** Introduce a session layer above the existing single-document model: `DocumentTab` owns per-file state, while `DocumentSessionViewModel` coordinates tabs, workspace root, dirty prompts, and selected document operations. Keep rendering, file I/O, workspace file management, outline parsing, appearance, export, and app-icon packaging behind focused services so SwiftUI views remain composition glue.

**Tech Stack:** Swift 5.9+, macOS 14+, SwiftUI, AppKit, WebKit/WKWebView, SwiftPM, XCTest when available, local app bundle script.

---

## Scope notes

V0.1.1 is a large integrated UI/state upgrade. The implementation must land in small commits and preserve V0.1.0 behavior after each stage where possible.

Do not add:

- Git integration
- plugin system
- cloud sync
- accounts
- collaboration
- full-text search
- App Store/signing/notarization workflow
- syntax highlighting
- complex theme marketplace
